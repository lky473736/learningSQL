<entity-relationship 모델>
		entity : 개체 (고유한 특성을 가진 것)
		attribute : 개체의 속성 (개체만이 가지고 있는 고유한 속성)
		ex) 환자정보 테이블 : 개체, 테이블의 row값 : 속성
	
key : 테이블 간에 데이터를 유일하게 공유할 수 있는 속성
key를 이용해서 두 테이블간 정보를 자유롭게 열람 가능
	
파일 시스템 : 응용프로그램마다 필요한 데이터를 별도의 파일로 관리 (비효율적)
DBMS : 데이터베이스 관리 시스템으로 일괄적으로 관리 (효율적)
	
RDB : 관계형 데이터베이스
	- table : 2차원 테이블 구조로 저장한 것 (relation)
	- attribute : column을 의미 (속성)
	- tuple (row) : row를 의미
	하나의 테이블의 tuple은 유일. 동일한 tuple이 2개 이상 작성 x
		
구조화된 틀 : 테이블이름(attribute 이름들)
ex) 주식창 (주식이름, 주가, 최저점, 최고점...)
